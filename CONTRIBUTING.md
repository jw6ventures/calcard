# Contributing to CalCard

Thanks for your interest in contributing to CalCard!

This project is open source under the GNU AFFERO GENERAL PUBLIC LICENSE also known as AGPL. We welcome contributions that improve the project while aligning with its goals and direction.

---

## Guidelines

- Keep changes focused and minimal
- Prefer clarity over cleverness
- Follow existing patterns in the codebase
- Discuss significant changes before implementing them


## Ways to Contribute

- Report bugs
- Suggest features
- Improve documentation
- Submit pull requests

For significant changes, please open an issue first to ensure alignment.


## Development Setup

1. Fork the repository  
2. Clone your fork  
3. Create a new branch (e.g., `feature/my-awesome-feature`)
4. Make your changes
5. Test your changes
6. Commit and push
7. Open a pull request


## Pull Request Guidelines
- Keep pull requests small and focused
- Include a clear description of what changed and why
- Link related issues if applicable
- Ensure your code builds and passes tests
- Avoid unrelated changes in the same PR
- Ensure you understand and can explain the changes you are submitting
- We may request changes or decline contributions that do not align with the long-term direction of the project.

Pull requests that do not align with the project's direction may be closed.


## AI-Generated Contributions

Contributions that are predominantly generated by AI tools are welcome, but must be carefully reviewed, validated, and understood by the contributor.

We reserve the right to prioritize, review, or decline such contributions at our discretion. Pull requests that appear to be low-effort or insufficiently validated may be closed without detailed review.

## Code Style
- Follow existing code style and conventions
- Write clear, readable code
- Add comments where necessary, but prefer code that is self-documenting
